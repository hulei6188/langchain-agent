# 智能体配置面板 Coze 风格改版设计文档 (Orchestration Layout Redesign)

**文档状态**：已由用户确认设计规范  
**日期**：2026-05-30  
**作者**：Antigravity (AI Coding Assistant)  
**目标**：重构 Lingshu 智能体编辑页面的配置面板，参考 Coze 采用分类折叠和弹窗添加的形式，提升系统排版档次，优化用户绑定工具与知识库的体验。

---

## 1. 核心视觉与交互痛点
*   **平铺拥挤**：原本的可用工具（如 Web 搜索、天气、计算器等）以及所有创建的知识库直接在主面板平铺展示，导致页面高度过大，配置主次不分，容易让用户感到混乱。
*   **样式生硬**：绑定的工具行和知识库状态不够精致，无法体现 Premium 智能体开发平台的质感。
*   **折叠功能单一**：仅有简单的 Panel 卡片包裹，缺乏像 Coze 一样的多分类流畅折叠交互。

---

## 2. 改版方案架构 (Proposed Architecture)

### 2.1 编排主列 (Builder Panel)
我们将原有的 `persona-panel`（左侧）和 `builder-panel`（中间）中涉及“编排配置”的部分按大类划分为流畅折叠的手风琴式面板：

1.  **技能大类 (Skills)**
    *   **子项：工具 (Tools)**
        *   **头部右侧**：显示精美的文本按钮 `+ 添加`（取代单调的 `+`）。
        *   **展开逻辑**：默认展开。仅显示**当前已绑定**到该智能体的工具（以精致卡片形式展示）。若没有绑定工具，展示微弱淡雅的“暂无绑定工具，点击添加”占位符。
        *   **卡片设计**：每个绑定工具作为一个独立卡片，包含高辨识度的彩色图标、加粗工具名、一行简短描述、运行密钥脱敏状态，以及鼠标悬停时右侧显现的 `✕` 解绑按钮。
        *   **收起状态**：隐藏绑定的工具卡片，仅在头部展示绑定的数量，如 `工具 (2)`。

2.  **知识大类 (Knowledge)**
    *   **子项：文本 (Texts / 知识库)**
        *   **头部右侧**：显示文本按钮 `+ 添加`。
        *   **展开逻辑**：默认展开。仅显示当前已绑定的知识库精致卡片（大小、切片数量、`✕` 解绑按钮）。
        *   **上传与文件管理区精简**：原本冗长的知识库下拉框、文档列表和拖拽上传区域，只有在用户点击选中某个已绑定卡片时，才在该卡片下方优雅地展开，大幅精简主面板的视觉高度。
        *   **收起状态**：隐藏卡片及上传管理区，仅头部展示数量，如 `文本 (1)`。

3.  **记忆大类 (Memory)**
    *   **子项：会话记忆 (Session Memory)**
        *   **展开逻辑**：折叠面板，展开后显示“会话记忆”启用开关，当开启时，显示数字消息上限滑动条（0.0 ~ 2.0 温度滑动条及上限均在这里操作）。
    *   **子项：用户记忆 (User Memory)**
        *   **展开逻辑**：折叠面板，展开后显示“运行时注入”开关、Facts 数据表、Preferences 偏好配置等多行编辑项。

4.  **对话体验大类 (Chat Onboarding)**
    *   **展开逻辑**：折叠面板，包含“开场白文案”多行输入框、以及“引导预置问题”的动态输入增删列表。

### 2.2 资源检索与绑定浮窗 (Portal Modals)

我们不合并弹窗，而是采用专注的**分离式轻量浮窗**：

#### 1. “选择工具”浮窗 (Select Tool Modal)
*   **触发**：点击工具栏的 `+ 添加`。
*   **结构**：
    *   **搜索栏**：顶部全宽的轻量搜索框，提供实时 React 状态过滤。
    *   **列表区**：垂直滚动的可用工具列表。
    *   **工具行卡片**：比左侧的卡片略宽，包含完整描述。
    *   **状态操作**：
        *   若未绑定：显示蓝色 `添加` 按钮。点击后调用 `toggleTool` API 进行绑定，且该行状态变成 `已添加` 禁用状态。
        *   若已绑定：直接显示置灰的 `已添加`。
*   **解绑**：随时在主面板的工具卡片上点击 `✕` 触发解绑。

#### 2. “选择知识库”浮窗 (Select KB Modal)
*   **触发**：点击知识库的 `+ 添加`。
*   **结构**：
    *   **顶部栏**：左侧为知识库搜索框，右侧为绿色的 **`+ 新建知识库`** 按钮（点击后直接弹出新建表单，创建成功后无缝带回选择列表）。
    *   **列表区**：可用知识库列表，展示其元信息（容量、文件数）。
    *   **状态操作**：未绑定的展示 `添加` 按钮，点击调用 `toggleKb` 进行绑定；已绑定的置灰展示 `已添加`。

## 3. 技术实现细节与数据流 (Data Flow)
*   **单一数据源**：所有展开/折叠状态（`expandedSections`）在 `BuilderView.jsx` 中以 React `useState` 数组或对象进行维护，确保状态纯净且由数据驱动。
*   **解耦绑定数据**：
    *   `agentForm.tool_ids` 保存已绑定的工具 ID。
    *   `agentForm.knowledge_base_ids` 保存已绑定的知识库 ID。
*   **弹窗管理**：
    *   `[toolsModalOpen, setToolsModalOpen]`：控制选择工具弹窗显示。
    *   `[kbModalOpen, setKbModalOpen]`：控制选择知识库弹窗显示。
*   **过渡动画**：通过 CSS Transitions 实现折叠和弹窗遮罩展开的柔和渐变，保证交互的丝滑感，完全对标 Premium 质感。

## 4. 验证与发布方案
*   **Vite 编译验证**：修改完成后，必须在前端目录下运行 `npm run build` 确保 TypeScript/React 代码无编译警告或错误。
*   **功能自测**：
    1.  验证工具与知识库的折叠/展开功能。
    2.  验证点击 `+ 添加` 弹出浮窗、模糊搜索功能、点击添加完成绑定。
    3.  验证点击卡片右侧 `✕` 快捷解绑功能。
    4.  测试深色与浅色模式，确保文本字字清晰，无色差异常。
