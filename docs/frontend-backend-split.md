# Frontend Backend Split

v2 默认就是前后端分离。

## 前端

React + Vite，单页应用内分为主页面和配置工作台：

- GPT 风格主页面：左侧导航和会话，中央聊天入口，聊天区展示来源引用。
- 智能体列表是主页面中的一个功能入口，创建/编辑智能体时才进入配置工作台。
- Coze 风格配置工作台：左侧人设与回复逻辑，中间模型设置、技能、知识、记忆、变量和对话体验，右侧预览与调试。
- 当前前端不暴露内部运行节点；后端默认链路仍负责实际执行。

## 后端

FastAPI 提供稳定 API。前端不直接访问 Python 模块、数据库或向量库。

## 边界

- 前端负责交互和状态展示。
- 后端负责认证、权限、默认运行链路、知识库索引和模型调用。
- PostgreSQL/Milvus/Redis 不暴露给浏览器。
- 前端提交 `mode`、`variables`、`rag_enabled` 和附件 id，草稿/发布选择、变量合并、RAG 默认值解析、记忆读取和运行执行都由后端完成。
- 前端展示“我的模型”配置入口，但模型 API key 只在创建或替换时提交给后端；后端只回传 `has_api_key`，不回显明文。
- 前端不再用图片探测结果拦截聊天发送。图片上传、粘贴、预览和发送属于聊天输入能力；如果模型或网关不支持图片，由后端转发后的真实上游响应决定。模型测试入口放在“我的模型”编辑弹窗里，只作为连接和图片请求诊断。
- 文档附件仍可被前端/后端按 `supports_document=false` 拦截，因为文档解析是平台本地能力，不是模型网关自证能力。
